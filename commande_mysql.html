Correction ... Se connecter avec son nom d'utilisateur(-u)

mysql -u root

ou ajouter l'option(-p) s'il y a un mot de passe

mysql -u root -p

2) Afficher la liste des BD ?

show databases ;

3) se rendre sur une base de données ? Si la base s'appelle sf6(par exemple)
use sf6
4) Afficher les tables ? N'oublier pas le (s) dans tables
show tables;
5) Afficher les colonnes ? i fo préciser quelle table

6) Commencer l'exploitation ! Afficher toutes les lignes de la table personne

Ajouter la table produits

rename table produits to ventes;

Ajouter l'utilisateur Toto

Changer le mot de passe de Toto
SET PASSWORD [ FOR 'Toto'@'localhost'] = PASSWORD( 'nouveau_mdp' ) ;

Afficher les droits de Toto( si host est le nom du serveur)
SHOW GRANTS FOR 'Toto'@'host' ;

⏰ Evolution CLI
La CLI bin/console est dépassée.
Dans Symfony6 on utilise une nouvelle CLI qui s'appelle symfony. Tout simplement!

Pour ce faire, il faut au préalable, installer un scoop, via le terminal Powershell à l'aide des 2 commandes suivantes:

1)Set-ExecutionPolicy RemoteSigned -scope CurrentUser
2)iwr -useb get.scoop.sh | iex

Ainsi, on peut utiliser ce scoop pour installer la nouvelle CLI

3) scoop install symfony-cli

⏰ Evolution Serveur
Àpartir de Symfony5, un serveur web est couplé au moteur Symfony.
Du coup, pas besoin d'installer un serveur comme dans Symfony4
⏰ Evolution Serveur
Au démarrage du serveur, l'option -d permet de faire tourner le serveur en arrière plan.
symfony server:start -d
On peut toujours arrêter le serveur avec la commande: Ctrl + C
