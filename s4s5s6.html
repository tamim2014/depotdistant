Les 3 dernières versions Symfony

Symfony4 Symfony5 Symfony6
Préalables • Installer scoop(via terminal powershel):
  Set-ExecutionPolicy RemoteSigned -scope CurrentUser
  iwr -useb get.scoop.sh | iex
• Installer la nouvelle CLI:
  scoop install symfony-cli
Importer un modèle de projet composer create-project modelDuProjet monProjet
Le model du projet est symfony/website-skeleton		 symfony new monProjet --full

On peut simplifier ainsi, à condition d'avoir pré-installer un microservice(console application ou API).
Sinon il faut mentionner --webapp 		• Utilisation de Symfony CLI:
  symfony new --webapp monProjet ou bien
  symfony new --webapp monProjet --version=6.0.*
• Utilisation de Composer:
  composer create-project symfony/skeleton:"6.1.*" monProjet
  cd monProjet
  composer require webapp
Installer un serveur composer require server - -d Àpartir de Symfony5, un serveur web est couplé au moteur Symfony.
Du coup, pas besoin d'installer un serveur comme dans Symfony4 		Serveur integré
Démarrer le serveur php bin/console server:run symfony server:start ou encore
symfony server:start --port numeroDePort ou bien
symfony serve 		symfony server:start ou encore
symfony server:start -d en arrière plan
Ajouter un contrôleur php bin/console make:controller Idem Symfony4 Idem Symfony4
Ajouter une BD parametrer la variable DATABASE_URL dans le fichier.env
php bin/console doctrine:database:create 		Idem Symfony4 Idem Symfony4
Ajouter les tables de la BD php bin/console make:entity
php bin/console make:migration
php bin/console doctrine:migrations:migrate 		Idem Symfony4 Idem Symfony4

Remarques:
    La CLI bin/console est dépassée.
    Dans Symfony6 on utilise une nouvelle CLI qui s'appelle symfony. Tout simplement!
