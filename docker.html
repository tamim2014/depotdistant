Lancer un conteneur
  Démarrer l’image ubuntu:latest:

Syntaxe de base
 • docker run
 --name nomConteneur
 -it nomSystèmeDexploitation

Options avancées
 • --rm: Supprime le container avant de le relancer.
 • -v: Volume de données
 • -it: Terminal interactif
 • -itd: Éxecute en arrière plan
 • -itp ou -it -p: Affectation de port
 • -p 3000:80 Déclare le port 80 du conteneur sur le port 3000 de votre machine.
 • ${PWD} Dossier couran(de ton PC)
 • ${PWD}:/data Monte le dossier courant dans le dossier /data du conteneur

 • Note: Sur Windows, remplacer
  -v ${PWD}:/data  par
  -v "C:\Data":/data

  Exemple:

# *nix
docker run --rm --name monConteneur -it -p 3000:80 -v ${PWD}:/data ubuntu:latest

# Windows
docker run --rm --name monConteneur -it -p 3000:80 -v "C:\Data":/data ubuntu:latest

Docker Compose(docker-compose.yaml)

Structure du fichier

C'est juste un fichier YAML

version: X.Y
services:
 nomService(exple mysql):
  image:
  ports:
  environment:
  volumes:
  profile:

Fin du fichier. Mais on a oublié un truc. il faut préciser la valeur de chaque paramètre. Pour un service mysql on peut faire par exemple:

  image: mysql:8.0
  ports:
   - "3306:3306"
  environment:
   - MYSQL_ROOT_PASSWORD=
   - MYSQL_ALLOW_EMPTY_PASSWORD=true
   - MYSQL_USER=petclinic
   - MYSQL_PASSWORD=petclinic
   - MYSQL_DATABASE=petclinic
  volumes:
   - "./conf.d:/etc/mysql/conf.d:ro"
  profiles:
   - mysql

Là c'est bon. Pour construire mon image docker, il suffi de lancer ce fichier avec la commande suivante:

  docker-compose up
